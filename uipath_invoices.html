A small back office helper built in UiPath Studio that scans a folder of text files, classifies them based on content, and automatically emails only the invoices to a chosen recipient. It removes manual triage work and makes sure no invoice is missed.

Project Summary

  • Goal: Automate invoice detection in text files and forward them by email to a selected address.
  • Tools: UiPath Studio - File System activities - String operations - SMTP Mail.
  • Inputs: Local folder path and recipient email address typed by the user.
  • Logic: Loop through each .txt file, read content, check for the keyword "Invoice", then send matching files as attachments.
  • Outputs: Only invoice files are emailed and each decision is logged for traceability.
Scope - practice project for UiPath and simple text based classification.

Workflow

A simple but realistic automation that could sit next to any small accounting process.

üü¢ Ask - what should be automated

The target scenario was an employee who regularly receives text files in a shared folder and has to manually identify which ones are invoices and forward them by email.

üßπ Prepare - inputs and rules

Define input parameters - folder path and recipient email - and a simple classification rule based on the presence of the keyword "Invoice" in the file content.

üîß Build - UiPath workflow

Use Browse For Folder to select the directory, then For Each File in Folder filtered by *.txt. Read Text File for each item, evaluate the condition, and branch into email or ignore path.

üìä Decide - invoice or not

If FileOutput.Contains("Invoice") is true, the file is treated as an invoice. Log messages tag each file as "Emailing invoice" or "Ignoring receipt" so runs are easy to audit.

üì§ Share - send by email

Use SMTP Mail with a dynamic subject built from the file name and attach the current file through a file resource. This mirrors how a human would attach and send each invoice.

üß† Extend - next ideas

The same pattern can be extended with stronger text classification rules, attachment formats other than txt, or integration with an RPA queue to track which invoices were processed.
